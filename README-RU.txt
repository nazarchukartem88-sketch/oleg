ГОТОВЫЙ САЙТ С СОБСТВЕННЫМ СЕРВЕРОМ И POSTGRESQL
=================================================

Что работает после размещения на Render:
- заявка с телефона сразу появляется в админке на компьютере;
- защищенный вход по email и паролю;
- заявки хранятся в общей онлайн-базе PostgreSQL;
- добавление объектов и до 8 фотографий;
- ссылка на видео;
- статусы и удаление заявок;
- данные одинаковые на всех устройствах.

ДАННЫЕ ВХОДА ПО УМОЛЧАНИЮ
Email:  admin@oleg-realtor.com
Пароль: OlegPanel-2026!

ВАЖНО: после запуска смените ADMIN_EMAIL и ADMIN_PASSWORD в Environment Render,
затем сделайте Manual Deploy / Restart. Пароль не показывается на сайте.

ПОШАГОВЫЙ ЗАПУСК
1. Распакуйте архив.
2. Создайте новый пустой репозиторий на GitHub.
3. Загрузите в него ВСЕ файлы из этой папки.
4. Откройте Render и выберите New + -> Blueprint.
5. Подключите GitHub и выберите репозиторий.
6. Render прочитает render.yaml и предложит создать:
   - Web Service oleg-realtor-site;
   - PostgreSQL oleg-realtor-db.
7. Нажмите Apply / Deploy и дождитесь статуса Live.
8. Откройте выданный адрес вида https://oleg-realtor-site.onrender.com
9. Админка: https://ВАШ-АДРЕС.onrender.com/office-ov

ПРОВЕРКА
1. Откройте сайт с телефона и отправьте заявку.
2. Откройте /office-ov на компьютере.
3. Войдите по данным выше.
4. Откройте вкладку «Заявки» — обращение должно появиться.

ОГРАНИЧЕНИЯ
- Бесплатный веб-сервис Render может засыпать и первый запуск занимает некоторое время.
- Фотографии хранятся в PostgreSQL, поэтому рекомендуется сжимать их перед загрузкой.
- Одна фотография — до 5 МБ, максимум 8 фотографий на объект.
