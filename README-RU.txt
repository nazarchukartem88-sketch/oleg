ГОТОВАЯ ВЕРСИЯ САЙТА РИЕЛТОРА — БЕЗ SUPABASE

Что работает:
- сайт и мобильная версия;
- заявки с телефона сохраняются в общей PostgreSQL-базе;
- заявки видны в админке на компьютере;
- вход администратора через собственный сервер;
- добавление и редактирование объектов;
- загрузка до 8 фотографий по 5 МБ;
- ссылка на видео YouTube/MP4;
- изменение контактов сайта;
- смена пароля администратора;
- экспорт заявок в CSV.

ВАЖНО: данные Supabase здесь вообще не используются.

Админка:
https://ВАШ-САЙТ.onrender.com/office-ov

Данные входа берутся из переменных ADMIN_EMAIL и ADMIN_PASSWORD в Render.
Для уже созданного ранее сервиса используйте значения, которые были указаны при первом запуске.

После первого входа сразу откройте «Налаштування» и измените пароль.

Если создаёте Blueprint заново, Render попросит ADMIN_EMAIL и ADMIN_PASSWORD. Введите свои значения и запишите их.

Структура архива упрощена: все файлы загружаются в корень GitHub-репозитория. Папка public не нужна.
